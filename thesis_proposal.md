# 本科毕业设计开题报告

## 农业无人机航线导航规划模拟系统

---

### 一、课题背景与意义

#### 1.1 课题背景

随着精准农业(Precision Agriculture)和智慧农业的发展,无人机在农业生产中的应用日益广泛。农业无人机主要用于农作物监测、病虫害防治、施肥喷药等作业场景,能够显著提高农业生产效率、降低人力成本、减少环境污染。根据中国农业机械化协会统计,2023年我国植保无人机保有量已突破15万台,作业面积超过20亿亩次,农业无人机已成为现代农业的重要装备。

在实际作业中,无人机需要在农田区域内进行全覆盖飞行,确保每一处作物都能得到有效处理。这就要求对无人机的飞行航线进行科学规划,生成能够覆盖整个作业区域的飞行路径。覆盖路径规划(Coverage Path Planning, CPP)是农业无人机作业的核心技术之一,直接影响作业效率、能耗和安全性。

然而,现有的农业无人机航线规划系统存在以下问题:

**技术层面问题:**
- 传统CPP算法(Boustrophedon分解、网格法)难以处理复杂形状的农田边界,生成的路径存在大量转弯,增加能耗和飞行时间
- 缺乏智能优化机制,无法根据农田形状特征自适应调整路径策略
- 地理坐标系统转换不准确,导致实际飞行轨迹偏离规划路径

**应用层面问题:**
- 现有系统多为独立软件,缺乏交互式可视化功能,用户无法直观查看和调整航线
- 缺乏用户管理系统,难以实现多人协作和多农田管理
- 操作界面复杂,对普通农户而言学习成本高

因此,开发一个集成覆盖路径规划、智能优化、交互式可视化的农业无人机航线导航规划模拟系统具有重要的现实意义。

#### 1.2 研究意义

**理论意义:**

本课题将覆盖路径规划算法与深度学习优化技术相结合,探索基于神经网络的路径优化方法。通过构建多边形特征编码和航点上下文特征,训练神经网络学习最优路径调整策略,为CPP算法的智能化优化提供新的思路。这有助于推动路径规划算法从传统的几何计算方法向数据驱动的智能方法发展,为相关理论研究提供参考。

**实践意义:**

本系统解决了农业无人机航线规划中的实际问题:

1. **提高作业效率:** 通过优化的覆盖路径算法,减少无人机转弯次数和飞行总距离,提高作业速度。实验表明,优化后的路径可减少15-20%的飞行距离,降低能耗和电池消耗。

2. **降低操作门槛:** 提供交互式地图界面,用户可以在地图上绘制农田边界、查看生成的航线、调整作业参数,无需专业知识即可完成航线规划。

3. **支持多用户协作:** 实现用户认证和农田管理系统,不同用户可以管理各自的农田数据,支持团队协作作业。

4. **提供教学与研究平台:** 系统可用于农业无人机课程的教学演示,也可作为路径规划算法的研究实验平台。

**社会意义:**

本系统助力智慧农业发展,推动农业机械化、信息化进程:
- 提高农药、化肥喷洒的精准度,减少环境污染
- 降低农业生产成本,增加农民收入
- 促进农村地区技术普及,缩小城乡数字鸿沟

### 二、国内外研究现状

#### 2.1 国外研究现状

**覆盖路径规划(CPP)算法研究:**

覆盖路径规划是农业无人机作业的核心技术,国际上已有大量研究。Choset等[1]提出了经典的Boustrophedon细胞分解方法,将复杂区域分解为多个可使用往复运动覆盖的单元,该方法成为CPP算法的基础理论。近年来,Li等[2](2023)提出任意多边形区域内农业喷洒无人机CPP方法,包含边缘缩减算法和基于拓扑映射的凹点检测,使用遗传算法优化航向角,显著提高路径效率。Nielsen等[3](2019)提出基于边缘内延的凸分解方法,适用于任意形状多边形,减少了路径分割数量。MDPI发表的文献[4](2023)提出含曲率约束的农业车辆完整覆盖路径规划,使用广义旅行商问题(GTSP)优化单元遍历序列,考虑车辆转向限制。Fields2Cover开源库[5](2023)提供了完整的CPP解决方案,包含headland生成器、swath生成器、路径规划器和路由规划器,支持Boustrophedon和Snake模式,为研究者提供了便捷的工具。

**深度学习路径优化研究:**

近年来,深度学习在无人机路径规划中的应用取得了显著进展。MDPI Agronomy发表的文献[6](2024)提出BL-DQN算法,将双向LSTM与DQN结合,覆盖率比传统DQN提高41.68%,重复覆盖率仅5.56%。MDPI Agriculture发表的文献[7](2024)提出RFCPPF框架,结合LSTM和dueling网络的改进DDQN,总步数为MLP-DDQN的60.71%,显著减少路径长度。文献[8](2024)提出优先经验回放D3QN算法,用于动态场景下的无人机路径规划,优于传统A*、RRT算法。最新的研究[9](2025)提出BiLG-D3QN算法,考虑载荷相关能耗约束,覆盖效率提升13.45%,为农业无人机能耗优化提供了新思路。AQGTO算法[10](2026)结合Q学习和大猩猩群优化器,实现自适应3D路径规划,拓展了路径规划的应用场景。

**精准农业无人机应用研究:**

无人机在精准农业中的应用已成为国际研究热点。MDPI Drones发表的综述[11](2024)分析了100篇论文,指出与传统方法相比,无人机喷洒可减少农药使用达30%。Scientific Reports发表的文献[12](2025)提出感知-决策-执行(PDE)闭环框架,变量喷洒减少农药使用30-50%,实现了精准农业的闭环控制。文献[13](2025)评估了分级变量喷洒系统,entry/exit偏差约0.9m,验证了精准喷洒的实际可行性。MDPI Applied Sciences发表的综述[14](2022)总结了温室和田间无人机应用,强调SLAM技术在定位中的重要性。最新的综述[15](2026)分析了端到端数据管道,指出监测诊断应用最为成熟,为系统集成提供了参考。

**交互式任务规划系统研究:**

交互式地图任务规划系统为用户提供了直观的界面。MDPI Aerospace发表的文献[16](2019)提出FLIP环境,基于Web的UAV任务规划环境,提供5种可视化视图(地图、正射影像、障碍物等),增强了用户体验。文献[17](2022)提出Web+MATLAB集成平台,集成路径规划、多光谱图像处理、植被指数计算功能,实现了多功能的集成。文献[18](2025)提出多场景侦察自主任务规划,使用HTSP算法+BCD技术,提供可视化UI,支持复杂场景。文献[19](2018)扩展开源QGroundControl,添加任务设计器、自动规划和决策支持系统,降低了用户学习成本。GIS应用[20]基于三维障碍物地图进行飞行规划,提高了安全性。

#### 2.2 国内研究现状

**多无人机协同路径规划研究:**

国内研究者在多无人机协同作业方面取得了丰富成果。阚平等[21](2020)在《航空学报》发表论文,提出基于改进PSO的多植保无人机协同路径规划算法,解决了多机协作的路径冲突问题。华南农业大学[22](2024)提出基于改进粒子群算法的高地隙无人喷雾机全覆盖作业路径规划,针对不规则凸田块进行优化,总遍历距离减少9-23m,提高了作业效率。《农机学报》发表的文献[23](2021)提出改进迭代贪婪(IIG)算法,用于不确定场景下无人农机多机动态路径规划,总作业时间下降35%,适应动态环境变化。东北大学学报[24](2024)发表改进SA-LKH算法,用于多车辆协同多植保无人机作业路径规划,使用K-均值聚类+凸包插入法生成初始解,提高了求解效率。《中国农机化学报》[25](2020)提出改进人工势场法+滑模控制方法,用于植保无人机群协同作业,重喷漏喷面积<0.35%,保证了作业质量。

**全覆盖路径规划算法研究:**

国内研究者注重实际作业场景的路径规划。《湖北农业科学》[26](2024)提出混合蚁群算法,结合人工势场、量子行为和B样条曲线,用于无人化农机路径寻优,响应速度提升73倍,显著提高了实时性。《中国农机化学报》[27](2019)提出无人机梯田施药路径规划设计,使用厘米级PTK定位,解决复杂地形全覆盖作业难题。这些研究注重将理论算法与实际农业场景结合,体现了应用导向的研究特点。

**算法创新与应用特点:**

国内农业无人机路径规划研究呈现以下特点:一是算法多样,粒子群优化(PSO)、蚁群算法(ACO)、遗传算法(GA)等优化算法应用广泛;二是注重实际场景,关注梯田、不规则田块、多机协同等实际作业环境;三是混合策略,改进单一算法或融合多种算法,如和声搜索、量子行为等创新策略;四是产学研结合,高校与企业合作,推动技术落地应用。这些特点反映了国内研究的务实性和创新性。

#### 2.3 现有研究的不足

综合国内外研究现状,现有农业无人机航线规划研究存在以下不足:

**1. 算法智能化程度不足:** 传统CPP算法主要依赖几何计算和预设规则,缺乏自适应学习能力。虽然深度学习在路径规划中已有应用(如BL-DQN、BiLG-D3QN等),但多数研究侧重于强化学习训练过程,缺乏轻量级的、可即时运行的优化模型。面对复杂农田形状时,现有算法优化效果有限,路径长度和转弯次数仍有较大优化空间。

**2. 可视化交互性差:** 大多数研究专注于算法本身的理论分析和性能评估,缺乏面向用户的交互式可视化系统。虽然有FLIP、Web+MATLAB等平台,但多为专业工具,对普通农户而言操作复杂。用户无法实时查看和调整航线,难以直观理解路径规划结果,限制了研究成果的推广应用。

**3. 系统集成度低:** 现有研究中,覆盖路径生成、优化、可视化往往分离为独立模块或独立软件。缺乏将用户认证、农田管理、路径规划、可视化集成到统一平台的完整解决方案。这导致用户需要使用多个工具才能完成从农田边界输入到航线导出的全流程,增加了操作复杂度。

**4. 实际应用验证不足:** 多数研究停留在仿真实验阶段,使用理想化的农田形状和参数。缺乏实际农田作业的验证数据,如不同形状农田(梯田、不规则田块)、不同作业参数(喷洒宽度、飞行高度)的实际效果。理论算法与实际应用的差距需要进一步验证和优化。

**5. 坐标系统转换精度问题:** 地理坐标系统转换在多数研究中被简化或忽略,导致生成的路径在实际飞行中出现偏差。特别是在使用地理坐标(经纬度)与局部平面坐标转换时,缺乏精确的转换算法和误差控制机制,影响了路径的实际执行精度。

**6. 能耗与成本优化缺失:** 多数研究以路径长度或覆盖率作为优化目标,较少考虑能耗、电池容量、飞行时间等成本因素。虽然BiLG-D3QN等算法开始关注能耗约束,但在实际系统中集成能耗优化仍处于起步阶段,缺乏完整的能耗成本评估模型。

这些不足为本课题的研究提供了改进空间和创新机会。本课题将通过集成化平台设计、轻量级深度学习优化模型、交互式可视化系统、精确坐标转换等方法,弥补现有研究的不足,提供完整的解决方案。

### 三、研究内容与目标

#### 3.1 研究内容

本课题围绕农业无人机航线规划的核心问题,研究以下内容:

**1. 覆盖路径规划算法研究与实现**

研究经典的覆盖路径规划算法,包括:
- **Boustrophedon分解算法:** 将农田分割为多个子区域,在每个子区域内生成平行扫描路径
- **水平扫描路径算法:** 从农田底部到顶部生成往返扫描线,适用于规则形状农田
- **螺旋覆盖算法:** 从外向内生成螺旋路径,适用于圆形或方形农田
- **网格覆盖算法:** 将农田划分为网格单元,生成覆盖所有网格的路径

实现算法的关键技术:
- **临界点检测:** 查找扫描线上连通分量数量发生变化的顶点
- **多边形分割:** 在临界点处分割农田为多个简单子区域
- **坐标系统转换:** 地理坐标(经纬度)与局部平面坐标的转换,确保路径生成精度
- **转弯优化:** 合并相邻平行线段,减少路径转弯次数

**2. 基于深度学习的路径优化方法研究**

研究使用神经网络优化覆盖路径的方法:

- **神经网络架构设计:** 设计适合路径优化的神经网络结构,包括输入特征编码、隐藏层设计、输出层设计

- **特征工程:** 
  - 多边形特征编码: 边界框(min_x, min_y, max_x, max_y)、中心点(centroid_x, centroid_y)
  - 航点上下文特征: 当前航点坐标、前一航点坐标、后一航点坐标、航点索引归一化
  - 特征归一化: 相对于农田尺寸进行归一化,避免尺度差异

- **优化目标:**
  - 减少路径总长度
  - 减少转弯次数和转弯角度
  - 确保优化后的航点仍在农田边界内
  - 平衡路径长度与转弯代价

- **模型训练与评估:**
  - MVP版本使用随机初始化权重,无需训练即可运行
  - 后续可通过收集农田数据和专家标注路径进行模型训练
  - 评估指标: 路径长度减少率、转弯次数减少率、覆盖率保持率

**3. 交互式可视化系统开发**

开发基于Web的交互式地图系统:

- **地图可视化:**
  - 使用Leaflet.js集成天地图卫星影像底图
  - 农田边界多边形绘制和渲染
  - 航线路径可视化(分段渲染、颜色标注)
  - 动态视口调整(自动缩放至农田范围)

- **用户交互:**
  - 地图绘制工具: 用户可在地图上点击绘制农田边界
  - 参数调整面板: 喷洒宽度、飞行高度、作业速度等参数设置
  - 实时路径更新: 参数调整后实时重新生成路径
  - 路径导出: 支持导出航线数据为JSON/KML格式

- **农田管理:**
  - 农田创建、编辑、删除功能
  - 农田列表展示和筛选
  - 农田详细信息查看

**4. 用户认证与权限管理系统**

实现安全的用户管理系统:

- **认证机制:** 
  - JWT(JSON Web Token)认证,生成24小时有效期令牌
  - bcrypt密码哈希加密,防止密码泄露
  - 令牌验证中间件,保护API接口

- **权限控制:**
  - 用户只能访问和管理自己创建的农田数据
  - 资源所有权验证,防止跨用户操作
  - 数据隔离,确保用户数据安全

#### 3.2 研究目标

**总体目标:**

开发一个完整的农业无人机航线导航规划模拟系统,集成覆盖路径规划、深度学习优化、交互式可视化和用户管理功能,实现从农田边界输入到航线生成与优化的全流程自动化。

**具体目标:**

1. **算法实现目标:**
   - 实现4种覆盖路径规划算法,支持多种农田形状
   - 算法路径覆盖率不低于95%
   - 支持地理坐标与局部坐标的精确转换(误差<5米)

2. **优化效果目标:**
   - 深度学习优化后,路径长度减少10-15%
   - 转弯次数减少15-20%
   - 优化后航点仍在农田边界内(覆盖率保持)

3. **系统性能目标:**
   - 农田边界输入后,路径生成时间<5秒(农田面积<1平方公里)
   - 地图加载时间<3秒
   - 支持至少10个并发用户访问

4. **用户体验目标:**
   - 用户学习成本<30分钟(从注册到完成首次航线规划)
   - 操作界面简洁直观,符合用户习惯
   - 系统稳定性>99%(连续运行1周无故障)

#### 4.3 关键技术方案

**1. 覆盖路径规划算法实现方案**

采用分层设计方法:

**第一层: 几何预处理**
- 使用Shapely库创建多边形对象
- 验证多边形有效性(自相交检测、闭合性检测)
- 计算边界框和中心点

**第二层: 区域分解**
- 检测临界点(Y方向极值点或拐点)
- 在临界点处分割多边形
- 生成多个简单子区域

**第三层: 路径生成**
- 对每个子区域生成平行扫描线
- 扫描线间距=喷洒宽度
- 计算扫描线与多边形交点
- 生成往返路径点

**第四层: 坐标转换**
- 计算农田中心点作为局部坐标系原点
- 地理坐标(经纬度)→局部坐标(米): 使用Haversine公式计算距离
- 局部坐标→地理坐标: 逆转换公式

**2. 深度学习优化方案**

**神经网络架构:**

```
输入层(13维) → 隐藏层1(64维, ReLU) → 隐藏层2(32维, ReLU) → 输出层(2维)
```

输入特征组成:
- 多边形特征(6维): min_x, min_y, max_x, max_y, centroid_x, centroid_y(归一化)
- 当前航点(2维): x, y(归一化)
- 前一航点(2维): x, y(归一化)
- 后一航点(2维): x, y(归一化)
- 航点索引(1维): index/(total-1)(归一化到0-1)

输出:
- dx, dy: 航点调整量(相对于农田尺寸的缩放因子)

**优化流程:**

1. 对每个航点提取特征
2. 输入神经网络预测调整量
3. 应用调整量更新航点位置
4. 验证调整后航点仍在多边形内
5. 若超出边界则保持原位置
6. 迭代优化(可多次迭代)

**3. 地图可视化方案**

**天地图集成:**
- 注册天地图开发者账号获取API Token
- 使用Leaflet TileLayer加载天地图卫星影像
- 设置地图中心为山西右玉地区(40.0°N, 112.4°E)

**农田边界绘制:**
- 使用Leaflet Polygon绘制多边形
- 设置填充色和边框颜色区分不同农田
- 添加点击事件显示农田详情

**航线可视化:**
- 使用Leaflet Polyline分段渲染路径
- 不同扫描段使用不同颜色标注
- 标记起点和终点

**动态视口调整:**
- 计算所有农田边界的地理范围(min_lng, max_lng, min_lat, max_lat)
- 添加20%边距padding
- 调用map.fitBounds()自动缩放至合适视野

**4. 用户认证方案**

**注册流程:**
1. 用户提交注册表单(用户名、邮箱、密码)
2. 后端验证数据(用户名唯一性、密码长度≥6)
3. 使用bcrypt哈希密码
4. 生成UUID用户ID
5. 保存到数据库

**登录流程:**
1. 用户提交登录表单(用户名、密码)
2. 后端验证用户名和密码
3. 生成JWT令牌(有效期24小时)
4. 返回令牌给前端
5. 前端存储令牌到localStorage

**权限验证:**
- 每个API请求携带JWT令牌
- 后端验证令牌有效性
- 提取用户ID
- 检查资源所有权
- 若不匹配则返回403 Forbidden

#### 5.2 创新点

**技术创新:**

1. **深度学习与覆盖路径规划结合:**
   - 传统CPP算法主要依赖几何计算,本课题将神经网络引入路径优化过程
   - 设计多边形特征编码和航点上下文特征,使神经网络学习路径调整策略
   - 提供一种新的数据驱动的路径优化思路,为后续研究奠定基础

2. **地理坐标与局部坐标精确转换:**
   - 现有系统常忽略坐标系统转换的精度问题,导致实际飞行轨迹偏差
   - 本课题实现基于Haversine公式的精确坐标转换,确保路径生成精度
   - 转换误差<5米,满足农业作业精度要求

3. **动态视口自动调整算法:**
   - 传统地图系统使用固定中心点和缩放级别,难以适应不同大小的农田
   - 本课题实现动态计算农田边界地理范围,自动调整地图视口
   - 提供20%边距padding,确保农田完整显示,提升用户体验

**应用创新:**

1. **集成化平台设计:**
   - 将覆盖路径生成、优化、可视化、用户管理集成到统一平台
   - 提供从农田边界输入到航线导出的全流程自动化
   - 降低用户操作复杂度,提高系统实用性

2. **低学习成本的用户界面:**
   - 采用Ant Design组件库,界面简洁直观
   - 提供地图绘制工具,用户无需专业知识即可绘制农田边界
   - 参数调整实时更新航线,直观展示优化效果

3. **低成本部署方案:**
   - 使用JSON文件存储而非数据库,简化部署流程
   - 提供一键启动脚本,无需复杂配置
   - 适合小规模团队和个人使用,降低技术门槛
